# 微信开发者工具打开指南

## 问题描述
微信开发者工具报错：`no found app.json`

## 问题原因
微信开发者工具需要 `app.json` 文件在项目根目录，但我们的项目结构是：
```
wx-travel-app/
├── miniprogram/     # 小程序代码目录
│   ├── app.js
│   ├── app.json
│   ├── app.wxss
│   └── ...
└── cloudfunctions/  # 云函数目录
```

## 解决方案

### 方案一：使用正确的项目根目录（推荐）
1. 打开微信开发者工具
2. 选择"导入项目"
3. 选择目录：`/Users/mac/Ryan/wx/wx-travel-app/miniprogram`
4. 输入AppID：`wx0ec08eca8627249b`（需要替换为你的小程序AppID）
5. 点击"导入"

### 方案二：使用项目配置文件
我已经在根目录创建了必要的配置文件：
- `app.js` - 小程序入口文件
- `app.json` - 小程序配置文件
- `sitemap.json` - 站点地图
- `project.config.json` - 项目配置

现在你可以：

## 配置步骤

### 1. 获取小程序AppID
1. 访问 https://mp.weixin.qq.com/
2. 注册小程序账号
3. 在"开发"->"开发设置"中获取AppID

### 2. 配置云开发环境
1. 在微信开发者工具中点击"云开发"
2. 开通云开发环境
3. 获取环境ID
4. 更新文件：
   - `miniprogram/app.js` 中的 `env` 配置
   - `cloudbaserc.json` 中的 `envId` 配置

### 3. 部署云函数
1. 在微信开发者工具中右键点击 `cloudfunctions` 目录
2. 选择"上传并部署：云端安装依赖"
3. 等待所有云函数部署完成

### 4. 初始化数据库
1. 在云开发控制台创建集合：
   - `users` - 用户信息
   - `routes` - 路线信息
   - `favorites` - 收藏关系
2. 设置集合权限（建议使用安全规则）

## 常见问题

### Q1: 仍然报错 "no found app.json"
A: 确保微信开发者工具版本是最新的，或者尝试重启开发者工具。

### Q2: 页面无法加载
A: 检查 `app.json` 中的页面路径是否正确，确保所有页面文件都存在。

### Q3: 云函数调用失败
A: 检查云开发环境配置，确保环境ID正确，云函数已部署。

### Q4: 地图功能无法使用
A: 需要在微信公众平台申请地图权限，并在 `app.json` 中配置。
